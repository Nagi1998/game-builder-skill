# 已批准 GDD 的网页游戏开发合同

本合同只把已明确批准的 GDD 变成可运行、可验证的儿童友好网页游戏。

## 进入门禁与权威输入

开始任何实现、依赖安装或游戏文件创建前，必须同时确认：

- 账本明确记录 `gdd_approved: true`；
- 输入是点名的已批准 `gdd_version`，页眉状态为 `已批准`；
- 开发计划逐项引用该版本的最小可玩范围和验收标准。

任一条件不满足就停止开发，返回 GDD 审阅。实现中发现规则不可执行、范围冲突或需要改变体验时，也暂停相关功能，回到 GDD 修改、递增版本和明确批准循环；不得静默改变已批准的体验。

## 技术与数据边界

| 情况 | 决定 |
| --- | --- |
| 默认或玩法较简单 | 使用原生 HTML、CSS、JavaScript；保持静态、易运行和易测试 |
| 需要重复碰撞检测或大量精灵 | 可以使用 Phaser，并在计划中说明原生方案为何更复杂或更难测试 |
| 需要多场景、物理，或集中管理动画与资源 | 可以使用 Phaser，并记录具体需求与带来的最小依赖 |
| 不满足框架条件 | 继续使用原生技术，不因个人偏好引入框架 |

首个版本默认不使用服务器、登录、联网多人、支付、广告、公开聊天、上传或追踪。只有新的 GDD 版本明确写出必要性、安全边界和验收标准，并再次获批，才能改变这些边界。

## 儿童内容安全实现边界

实现必须保留获批 GDD 的儿童安全边界，不得加入或细化可操作的现实危险、自残、仇恨式针对、性内容、赌博或真钱随机机制、战利品箱操纵、暗黑模式或剥削性变现。相关玩法只能采用虚构、非血腥、非性化、不歧视且不使用真钱的安全版本；任何会突破这些边界的实现请求都停止开发并返回安全的 GDD 审阅。

## 测试驱动开发循环

每一项 GDD 验收标准和每一个缺陷都遵循同一循环：

1. 先写对应的自动化验收测试；无法自动化的视觉或触控行为，先在 `game/PLAYTEST.md` 写出可重复的失败验证步骤。
2. 运行测试或验证步骤，看到由缺失行为导致的预期失败。
3. 只实现让该项通过的最小行为。
4. 重新运行并确认通过，同时确认已有测试仍为绿色。
5. 只在绿色状态下重构，再次运行验证。

没有先观察到预期失败，不得把测试记作开发证据。实现冲突要回到 GDD 审批，不能通过修改测试来掩盖。

## 默认交付结构

```text
game/
├── index.html
├── styles.css
├── src/
├── tests/
├── assets/
├── README.md
└── PLAYTEST.md
```

最终交付物必须包含：

- `game/` 内可运行的源代码和所需资源；
- `game/README.md`，说明本地运行方式、测试命令、浏览器要求和键盘/触控操作；
- `game/tests/` 内与每条 GDD 验收标准对应的自动化测试；
- `game/PLAYTEST.md`，记录人工与浏览器验证证据；
- 获批的最终 GDD、`gdd_version` 和批准状态。

## 无障碍与响应式要求

- 桌面与手机布局都能完成核心流程，内容不被遮挡或截断；
- 核心玩法可用键盘完成，焦点顺序合理且焦点样式清晰可见；
- 手机提供与键盘操作等价的触控控制，触控目标大小和间距适合儿童；
- 文字、控件与背景有清晰对比度；
- 不使用只能通过悬停触发的操作；
- 有明显移动、闪烁或持续动画时，响应减少动态效果的系统偏好，并提供适当的动效减弱。

## 运行时合同

游戏必须可开始；支持暂停或重新开始；能到达胜利与失败；对关键动作、进展、错误、胜利和失败给出清楚反馈。页面加载和完整核心流程中不得出现未处理异常或控制台错误。

## 浏览器验证

在交付前完成一次桌面视口和一次常见手机视口的完整试玩，并记录尺寸与浏览器：

1. 从加载开始走通开始、核心循环、暂停或重新开始、胜利和失败。
2. 在桌面验证键盘控制，在手机视口验证触控等价控制。
3. 检查布局、可见焦点、对比度、无悬停依赖和适用的动效减弱。
4. 打开并检查控制台，确认没有未处理异常或控制台错误。
5. 将获批 GDD 的每一条验收标准逐项与自动化或人工证据对照；没有证据的条目不得标记通过。

## `game/PLAYTEST.md` 记录格式

`game/PLAYTEST.md` 必须包含：

- 已批准 GDD 版本；
- 测试环境，包括浏览器、操作系统和日期；
- 运行与测试命令；
- 自动化测试结果；
- 人工试玩步骤与实际结果；
- 桌面和手机视口尺寸及各自结果；
- 键盘与触控等价控制结果；
- 控制台检查结果；
- GDD 验收标准到证据、结果的验收矩阵；
- 已知限制及其对体验的影响。

失败项必须记录可复现步骤，回到测试驱动循环修复并重新验证。

## 资源规则

只使用原创、程序生成、许可明确或已获授权的资源，并在 `game/README.md` 记录来源与许可。不得复制在世艺术家的独特风格，不得使用未经许可的商业图片、字体、音乐、音效或其他素材。无法确认许可时，改用原创或程序生成的占位资源。
