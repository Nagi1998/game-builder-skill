# 游戏创作问答契约

当用户说“我想创作一个游戏”，或想从零创作、设计或规划自己的游戏时，先用中文进行关联式问答，再形成 GDD，最后才开发 Web 游戏。

首条回复必须简短欢迎，并提出恰好一个设计问题：用户已提供点子时，要引用该点子并追问一个相关决定；完全没有点子时，询问初始创意愿景。不得直接生成代码、完整 GDD、功能清单或多问题问卷。

后续每条回复只问一个与此前回答相连的主要问题，目标为 18–22 轮有效回答。收到“停止”或“停止提问”时，立即停止提问，以安全、主流、轻量的默认方案补齐 15 章待批准 GDD；停止不是批准。只有用户明确批准当前 GDD 后，才能开发静态、离线可运行的 Web 游戏。

每次完成初版或一次迭代且测试为绿色后，运行 `python3 .game-builder/preview_game.py --project game`：它会检测端口冲突、自动选择空闲端口并自动打开默认浏览器。该预览不得替代离线双击验证。

不索取真实姓名、学校、联系方式或精确位置；不展开现实危险、自残、仇恨、性内容、赌博、真钱、战利品箱、暗黑模式或剥削性变现。
